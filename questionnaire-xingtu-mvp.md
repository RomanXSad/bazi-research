## Часть 2: Idea Stage — Решение правильное?

### 2.1 Наш продукт vs конкуренты

| Параметр | XingTu | Конкуренты |
|----------|--------|------------|
| Генератор имён EN→CN | ✅ Есть | ❓ |
| BaZi предсказание | ✅ Есть | ❓ |
| Привязка амулета к имени | ❌ Не готово | ❓ |
| Поведенческий трекер | ❌ Не готово | ❓ |
| Работает офлайн | ✅ Да | ❓ |
| Монетизация | ❓ Не решена | ❓ |

### 2.2 Три главных допущения

1. **Пользователь купит амулет** на основе сгенерированного имени
2. **Бесплатный генератор → трафик → конверсия** в платящих
3. **BaZi-предсказание даст retention** (возвращаемость)

### 2.3 А что если?

| Допущение неверно → | Последствие |
|---------------------|-------------|
| Никто не покупает амулет | Только ad-revenue? |
| Трафик есть, конверсия <0.5% | Модель не сходится |
| Пользователь пришёл 1 раз и ушёл | Нет LTV |

## Часть 5: Amulet Monetization — Воронка

### 5.1 Воронка конверсии (гипотезы)

```
Приход → Генерация имени → Интерес к амулету →
  → Просмотр амулета → Корзина → Оплата → Доставка
```

Где мы теряем больше всего? (нужны данные)

### 5.2 Бизнес-модель

| Параметр | Гипотеза | Нужно подтвердить |
|----------|----------|-------------------|
| CPM трафика | $5-15 | Анализ |
| Конверсия → генерация имени | 60% | Prototype |
| Конверсия → просмотр амулета | 10% | Prototype |
| Конверсия → покупка | 2-5% | MVP |
| Средний чек амулета | $15-30 | Исследование |
| Себестоимость (производство+доставка) | $5-10 | Поставщик |

### 5.3 Что измерять трекером

- Сколько дошли до генерации имени
- Сколько нажали "Получить амулет"
- Сколько кликнули на амулет в корзине
- Сколько начали оформление
- Сколько оплатили
- Retention D1 / D7 / D30
- Сколько вернулись за вторым предсказанием

## Часть 6: Roadmap

```
Этап 1 — Валидация гипотез (1-2 недели)
  ├── Customer discovery: 5-10 интервью
  ├── Опрос целевой аудитории
  └── Анализ отзывов конкурентов (app stores)

Этап 2 — MVP Scope + Build (2-3 недели)
  ├── Scope definition document
  ├── Amulet flow (каталог → корзина → оплата)
  ├── Поведенческий трекер (custom events)
  ├── Расширение базы имён до 200+
  ├── Security review перед deploy
  └── Payment integration (Stripe)

Этап 3 — Beta Launch (1 неделя)
  ├── 5-10 бета-тестеров
  ├── Measurement framework
  ├── Feedback loop
  └── Публикация на Product Hunt / Reddit

Этап 4 — Full Launch (2-4 недели после PMF)
  ├── Android APK → Play Store
  ├── Web → собственный домен
  ├── Content marketing (YouTube shorts)
  └── Итерации по отзывам
```
